# 018 게시글 검색 계획

## 설계 방향

게시글 검색은 기존 목록 조회 유스케이스를 확장한다. `keyword`가 있으면 제목 또는 본문에 키워드가 포함된 삭제되지 않은 게시글만 조회하고, 없으면 기존 전체 목록 조회를 유지한다.

## 애플리케이션 계층

- `ListPostsQuery`에 `keyword`를 추가한다.
- `keyword`는 `null` 또는 공백이면 검색 조건 없이 처리한다.
- 검색 결과도 기존 목록 조회와 동일하게 작성자 닉네임과 댓글 수를 배치 조회한다.
- 페이징 검증은 기존 `page`, `size` 규칙을 유지한다.

## 포트와 어댑터

- `PostRepositoryPort`에 키워드 기반 최신순 페이징 조회 메서드를 추가한다.
- Persistence 어댑터는 `title` 또는 `content` 대소문자 무시 포함 검색을 사용한다.
- 삭제된 게시글은 `deletedAt is null` 조건으로 제외한다.
- 검색어가 없는 경우 기존 전체 목록 조회 메서드를 그대로 사용한다.

## Web 어댑터

- `GET /posts`에 선택 파라미터 `keyword`를 추가한다.
- 응답 DTO와 성공 메시지는 기존 목록 조회와 동일하게 유지한다.
- Swagger 문서는 `keyword` 검색 조건을 한글로 설명한다.

## 테스트 전략

- 애플리케이션 서비스 테스트에서 검색어가 있는 경우 검색 포트를 호출하고 댓글 수를 포함하는지 검증한다.
- 애플리케이션 서비스 테스트에서 공백 검색어는 기존 전체 목록 조회로 처리되는지 검증한다.
- Persistence 어댑터 테스트에서 제목/본문 검색, 대소문자 무시, 삭제 게시글 제외를 검증한다.
- Web 어댑터 테스트에서 `keyword` 파라미터 전달을 검증한다.
- 구현 완료 후 `.\gradlew.bat test`를 실행한다.
