# 003 회원가입 명세

## 목표

사용자가 이메일, 비밀번호, 닉네임으로 회원 계정을 만들 수 있게 한다.

## 사용자 이야기

게시판 사용자는 자신의 계정으로 서비스를 이용하기 위해 회원가입을 하고 싶다.

## 범위

- 이메일, 비밀번호, 닉네임으로 회원을 생성한다.
- 이메일 중복을 검증한다.
- 비밀번호는 최소 길이를 검증한 뒤 암호화해서 저장한다.
- 회원가입 성공 시 생성된 회원 식별자와 공개 가능한 회원 정보를 반환한다.
- Swagger에 회원가입 API를 한글로 문서화한다.

## 비범위

- 로그인 API
- JWT 또는 세션 발급
- 이메일 인증
- 비밀번호 재설정
- 회원 정보 수정
- 게시글 작성자와 회원 계정 연결

## 도메인 규칙

- 이메일은 필수다.
- 이메일은 앞뒤 공백을 제거하고 소문자로 정규화한다.
- 이메일은 단순 이메일 형식이어야 한다.
- 이메일은 중복될 수 없다.
- 비밀번호는 필수이며 최소 8자 이상이어야 한다.
- 비밀번호 원문은 저장하지 않고 BCrypt 해시만 저장한다.
- 닉네임은 필수다.
- 닉네임은 앞뒤 공백을 제거하고 최대 50자까지 허용한다.
- 생성된 회원은 생성 시각을 가진다.

## API

```http
POST /auth/signup
Content-Type: application/json
```

요청:

```json
{
  "email": "minu@example.com",
  "password": "password123",
  "nickname": "minu"
}
```

성공 응답:

```http
201 Created
```

```json
{
  "id": 1,
  "email": "minu@example.com",
  "nickname": "minu",
  "createdAt": "2026-05-20T00:00:00Z"
}
```

중복 이메일 응답:

```http
409 Conflict
```

```json
{
  "code": "DUPLICATE_EMAIL",
  "message": "이미 가입된 이메일입니다.",
  "path": "/auth/signup",
  "timestamp": "2026-05-20T00:00:00Z",
  "errors": []
}
```
