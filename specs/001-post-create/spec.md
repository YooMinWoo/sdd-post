# 001 게시글 작성 명세

## 목표

클라이언트가 제목, 본문, 작성자를 입력해 게시글을 작성할 수 있게 한다.

## 사용자 이야기

게시판 사용자는 자신이 작성한 글을 저장하고 나중에 조회할 수 있도록 게시글을 작성하고 싶다.

## 범위

- 제목, 본문, 작성자로 게시글 1건을 생성한다.
- 저장 전 게시글 입력값을 검증한다.
- 생성된 게시글 식별자와 저장된 값을 응답한다.

## 비범위

- 인증과 인가
- 게시판 카테고리
- 첨부파일
- 댓글
- 좋아요
- 게시글 수정, 삭제, 목록, 상세 조회

## 도메인 규칙

- 제목은 필수다.
- 제목은 최대 100자까지 허용한다.
- 본문은 필수다.
- 본문은 최대 5,000자까지 허용한다.
- 작성자는 필수다.
- 작성자는 최대 50자까지 허용한다.
- 제목, 본문, 작성자는 검증 전에 앞뒤 공백을 제거한다.
- 생성된 게시글은 생성 시각을 가진다.

## API

```http
POST /posts
Content-Type: application/json
```

요청:

```json
{
  "title": "안녕하세요",
  "content": "첫 번째 게시글입니다.",
  "author": "minu"
}
```

성공 응답:

```http
201 Created
```

```json
{
  "id": 1,
  "title": "안녕하세요",
  "content": "첫 번째 게시글입니다.",
  "author": "minu",
  "createdAt": "2026-05-20T00:00:00Z"
}
```

잘못된 입력 응답:

```http
400 Bad Request
```
