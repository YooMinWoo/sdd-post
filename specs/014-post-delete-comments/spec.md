# 014 게시글 삭제 시 댓글 삭제 명세

## 목표

게시글 작성자가 게시글을 삭제하면 해당 게시글에 작성된 댓글도 함께 삭제되게 한다.

## 사용자 이야기

게시판 사용자는 자신이 작성한 게시글을 삭제할 때, 더 이상 접근할 수 없는 게시글의 댓글도 함께 정리되기를 원한다.

## 범위

- `DELETE /posts/{postId}` 성공 시 해당 게시글의 댓글을 함께 삭제한다.
- 댓글은 소프트 삭제가 아니라 물리 삭제로 처리한다.
- 게시글 삭제와 댓글 삭제는 같은 트랜잭션 안에서 처리한다.
- 게시글 삭제 API의 요청/응답 계약은 기존과 동일하게 유지한다.
- 삭제 성공 시 응답 본문 없이 `204 No Content`를 반환한다.
- 삭제된 게시글의 댓글 목록 조회는 기존 삭제 게시글 정책대로 `404 Not Found`, `code=POST_NOT_FOUND`를 반환한다.
- 댓글이 없는 게시글 삭제도 정상 성공으로 처리한다.

## 비범위

- 댓글 단독 삭제 API
- 댓글 소프트 삭제
- 댓글 삭제 이력 조회
- 댓글 복구
- 관리자 권한 기반 댓글 정리
- 게시글 삭제 API 응답 형식 변경
- 댓글 삭제 알림 또는 이벤트 발행

## 도메인 규칙

- 게시글 삭제 권한 검증이 성공한 경우에만 댓글을 삭제한다.
- 게시글이 없거나 이미 삭제된 게시글이면 댓글 삭제를 수행하지 않는다.
- 삭제 요청자가 게시글 작성자가 아니면 댓글 삭제를 수행하지 않는다.
- 게시글 id 또는 요청자 회원 id가 유효하지 않으면 댓글 삭제를 수행하지 않는다.
- 댓글 삭제 실패 시 게시글 삭제도 함께 롤백되어야 한다.

## API

기존 게시글 삭제 API를 그대로 사용한다.

```http
DELETE /posts/{postId}
Authorization: Bearer access.jwt.token
```

성공 응답:

```http
204 No Content
```

삭제 후 댓글 목록 조회:

```http
GET /posts/{postId}/comments?page=0&size=10
```

```http
404 Not Found
```

```json
{
  "success": false,
  "code": "POST_NOT_FOUND",
  "message": "게시글을 찾을 수 없습니다.",
  "data": null,
  "path": "/posts/1/comments",
  "timestamp": "2026-05-21T00:00:00Z",
  "errors": []
}
```
