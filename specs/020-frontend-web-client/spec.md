# 020 프론트엔드 웹 클라이언트 명세

## 목표

기존 게시판 API를 사용하는 별도 React 기반 웹 클라이언트를 제공한다. 사용자는 브라우저에서 회원가입, 로그인, 게시글 목록 조회, 검색, 상세 조회, 작성, 수정, 삭제, 댓글 작성, 수정, 삭제를 수행할 수 있어야 한다.

## 사용자 이야기

사용자는 API 문서나 별도 도구 없이 웹 화면에서 게시글을 탐색하고, 로그인 후 자신의 게시글과 댓글을 관리하고 싶다.

## 범위

- 별도 프론트엔드 저장소 `post-front`를 생성한다.
- React, Vite, TypeScript 기반 SPA로 구현한다.
- 게시글 목록, 게시글 상세, 게시글 작성, 게시글 수정, 로그인, 회원가입 화면을 제공한다.
- 댓글 목록, 댓글 작성, 댓글 수정, 댓글 삭제 기능을 게시글 상세 화면에서 제공한다.
- accessToken과 refreshToken은 프론트엔드 로컬 저장소에 저장한다.
- API 호출 시 accessToken을 Bearer 토큰으로 전달한다.
- 401 응답이 발생하면 refreshToken으로 토큰 재발급을 1회 시도한다.
- 백엔드는 프론트엔드 개발 서버 Origin을 허용하는 CORS 설정을 제공한다.

## 비범위

- 모바일 네이티브 앱
- PWA 설치 지원
- 소셜 로그인
- 이미지 업로드
- 관리자 기능
- 운영 배포 자동화
- HTTPS, 도메인 연결, Nginx reverse proxy 구성
- 쿠키 기반 인증 전환

## 도메인 규칙

- 비로그인 사용자는 게시글 목록, 게시글 상세, 댓글 목록을 조회할 수 있다.
- 로그인 사용자는 게시글과 댓글을 작성할 수 있다.
- 게시글 작성자 본인만 해당 게시글의 수정/삭제 버튼을 볼 수 있다.
- 댓글 작성자 본인만 해당 댓글의 수정/삭제 버튼을 볼 수 있다.
- 권한 최종 판단은 백엔드 API 응답을 기준으로 한다.

## API 연동 규칙

- API 기본 주소는 `VITE_API_BASE_URL` 환경변수로 설정한다.
- 로컬 기본 API 주소는 `http://localhost:8080`이다.
- 성공 응답은 기존 `ApiResponse<T>` 구조를 따른다.
- 실패 응답은 `code`, `message`를 화면 에러 메시지로 활용한다.
- `204 No Content` 응답은 별도 응답 본문 없이 성공으로 처리한다.

## UI 기준

- 첫 버전의 디자인은 흰색과 중립색 중심의 깔끔한 커뮤니티 게시판 톤을 사용한다.
- 모바일, 태블릿, 데스크톱에서 주요 화면이 깨지지 않아야 한다.
- 모든 주요 화면은 로딩, 빈 상태, 실패 상태를 제공한다.
