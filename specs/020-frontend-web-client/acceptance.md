# 020 프론트엔드 웹 클라이언트 인수 조건

## 인수 조건

- 사용자는 `/`에서 게시글 목록을 볼 수 있다.
- 사용자는 키워드로 게시글 목록을 검색할 수 있다.
- 사용자는 게시글을 선택해 상세 화면으로 이동할 수 있다.
- 사용자는 상세 화면에서 댓글 목록을 볼 수 있다.
- 사용자는 회원가입과 로그인을 할 수 있다.
- 로그인 사용자는 게시글을 작성할 수 있다.
- 로그인 사용자는 자신이 작성한 게시글을 수정하거나 삭제할 수 있다.
- 로그인 사용자는 댓글을 작성할 수 있다.
- 로그인 사용자는 자신이 작성한 댓글을 수정하거나 삭제할 수 있다.
- accessToken 만료로 401 응답이 발생하면 refreshToken으로 재발급 후 원래 요청을 재시도한다.
- refreshToken도 유효하지 않으면 로그아웃 상태가 된다.
- 로컬 프론트 개발 서버 `http://localhost:5173`에서 백엔드 API를 호출할 때 CORS 오류가 발생하지 않는다.

## 예외 케이스

- API 실패 응답이 오면 `message`를 사용자에게 표시한다.
- 게시글 목록이 비어 있으면 빈 상태를 표시한다.
- 댓글 목록이 비어 있으면 빈 상태를 표시한다.
- 비로그인 사용자가 글쓰기 화면에 접근하면 로그인 화면으로 이동한다.
- 비로그인 사용자가 댓글 작성을 시도할 수 없도록 입력 UI 대신 로그인 안내를 표시한다.

## 완료 판정

- 백엔드 테스트가 통과한다.
- 프론트엔드 테스트와 빌드가 통과한다.
- 로컬에서 주요 사용자 흐름을 수동으로 확인할 수 있다.
